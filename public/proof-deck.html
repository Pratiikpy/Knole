PROOF 01

Anonymised before the model

The single most load-bearing claim: the LLM never sees who you are. A local NER model masks every person, place, and organisation to stable tokens before any text leaves the box; a server-side reverse map restores your words in the reply. One gateway — no call site can forget to scrub.

npm run test:anon
WHAT YOU WROTE (encrypts under your key, stored on 0G):
  "Told Mara the real fear about moving to Berlin — that leaving the job at
   Google means I was never serious about the writing. Even Dr. Okafor noticed."

WHAT THE MODEL ACTUALLY RECEIVES (anonymised):
  "Told [PERSON_1] the real fear about moving to [PLACE_1] — that leaving the job at
   [ORG_1] means I was never serious about the writing. Even Dr. [PERSON_2]or noticed."

RESTORED IN THE REPLY (reverse map, server-side only):
  [PERSON_1]→Mara   [PLACE_1]→Berlin   [ORG_1]→Google   [PERSON_2]→Okaf
No full identifiable name reaches the model — gated statistically by the privacy-leak eval

The honest worst case is shown deliberately: [PERSON_2]or — the NER tagged only the prefix of "Okafor", leaving a meaningless or fragment. The identifiable name is still gone. NER is probabilistic, so the guarantee is "no full identifiable name reaches the model" (piiScrubRate ≥ 0.85), not "every byte of every name."

PROOF 02

Encrypted under your key on 0G

Your entry is AES-256-GCM ciphertext before it touches the chain, sealed under a key derived for you alone (HKDF-SHA256, your user id as the domain separator). Fetch the 0G blob with no key and there is nothing to read.

  your key ............... recovers the JSON envelope, obj.text === plaintext ✓
The 0G blob holds ciphertext only; nothing but your key recovers it

PROOF 04

Nobody else can read it

A journal is worthless if another user — or a forged request — can reach your inner life. Three personas with deliberately overlapping feelings run full journeys; every user is then probed with every other user's life.

npm run test:multiuser
  recalled own memory : true  ("You have concerns about your left knee…")
  recalled own memory : true  ("You're a new parent to a baby named Theo…")
  recalled own memory : true  ("You've started taking weekly piano lessons…")
  isolation — retrieval leaks: 0, ask leaks: 0
Every user's memory composes; no retrieval or ask ever crosses the user boundary

The engine-level data-isolation eval additionally proves IDOR: user A calling setMemoryStatus / updateMemoryContent on user B's memory id is a no-op.

PROOF 07 new

Sealed inference is on

The fourth privacy layer, now live. Reflection runs inside a 0G TEE — 0G Private Computer, model glm-5.1 — so even the operator can't read what the model writes back. A stalled enclave falls back to NVIDIA within 15s, and the name-scrub protects that fallback too, so the app never goes dark and the guarantee never weakens.

chatSealed("…a hard day", { maxTokens: 1200 }) · 0G Private Computer
POST router-api.0g.ai/v1/chat/completions   (stream: false)
  sealed .... true          ← served by the 0G TEE, not the NVIDIA fallback
  model ..... glm-5.1
  reply ..... "After such a heavy day, I hope you find a soft place to rest tonight."
The reflection came back from the 0G enclave — sealed = true, model = glm-5.1

sealedActive() gates the on-Mirror attestation badge, so the "composed where even we can't read it" claim only ever appears when the inference genuinely ran in the enclave — never on the fallback.

PROOF 08 new

You own it, as a token

Your memory isn't only yours to read — it's yours to hold. The Memory iNFT (ERC-7857) mints an encrypted snapshot of who you are to your own wallet; the token records only the 0G storage root and a hash. And it can never be taken from you or sold: every transfer path reverts in the contract.

KnoleMemory (ERC-7857) · 0G Galileo
  contract ............... 0xf45C33fa8005734E67F9E99De844D220A18D898E
  mint(to = your wallet) . Token #3 · encryptedURI = 0g://<root> · metadataHash set
  approve() .............. revert — not for sale
  setApprovalForAll() .... revert — not for sale
Minted to the user's own wallet on 0G; transfer is impossible by construction

Mint and self-custody need no oracle (the ERC-7857 oracle only re-encrypts on a sale, which Knole deliberately never does). evolve() re-anchors a fresh snapshot as your memory grows, keeping the same token.

DEPTH

Under the hood

Reflections, chat, Ask, the Mirror, and the nudge all draw on one engine — and one invariant: plaintext is encrypted before it touches the chain, anonymised before it touches the model.

THE MEMORY ENGINE

Embed · extract · reconcile · retrieve

Embed locally
all-MiniLM-L6-v2, 384-dim, no API call. embed.ts
Extract facts
an LLM reads the ANONYMISED text only — people, goals, patterns, commitments, values. engine.ts → sealed.ts
Reconcile
content-hash UPSERT dedup, then a 3-way LLM judge: reinforce a reworded duplicate · supersede a contradiction (bi-temporal: kept with invalid_at) · keep the independent. engine.ts
Encrypt + anchor
AES-256-GCM under your key → upload to 0G Storage → root anchored on the entry row. og.ts
Retrieve
RRF hybrid: pgvector cosine ⊕ lexical full-text, fused — an exact keyword surfaces even when the vector arm wouldn't. engine.ts

Recall earns importance (retrieving a memory bumps its recall_count); a user edit flips confidence to 1.0 and wins any contradiction; a forgotten memory never surfaces again, yet memory_history still records that it existed.

HONEST SCOPE

What it is, and what it isn't

Knole hides

  • Your entry text — AES-256-GCM ciphertext on 0G
  • The real names / places / people from the model
  • Your data, from us — sealed under your key

Knole does not hide

  • That an entry exists (the 0G anchor is public)
  • The on-chain root hash
  • That a testnet wallet signs the anchors

Pre-mainnet, stated plainly: the TEE "even we can't read it" path is on (0G Private Computer, glm-5.1); a stalled enclave falls back to NVIDIA within 15s, and the anonymise scrub protects that fallback too. The logged-in run is proven headlessly with a real inbox + Privy OTP. The rate limiter is in-memory; mainnet waits on an external audit and KMS-backed key custody. Full detail in SECURITY.md and docs/PROOF.md.
